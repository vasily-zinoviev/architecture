# Definition of Ready
## Критерии готовности задачи к тому, чтобы взять ее в работу (заполняется командой на PBR)
- Не зависит от других историй
- Понятна команде
- Приносит ценность для клиента
- Выполнима за 1-2 спринта
- Описание истории сформулировано в соответствии с шаблоном "Я, как пользователь, хочу (намерение), чтобы (цель)" в терминах ценности для пользователя
- Проведена аналитика по истории
- Архитектура описана в достаточном объеме для разработки, спецификации API выложены в репозиторий кода
- Проставлены имена ответственных (роли Разработчик, Тестировщик)
- Требования ясны и понятны всей команде
- Описаны функциональные и нефункциональные требования
- Дизайн интерфейса понятен и согласован командой
- Можно протестировать результат, тестовые сценарии ясны и понятны и зафиксированы
- История включена в спринт и в релиз
- Инфраструктура для развертывания готова и доступы получены
- История переводится в статус Открыто
# Definition of Done
## Критерии полной готовности задачи (заполняется по мере готовности командой)
- Разработчик создает подзадачи в истории и приступает к реализации (Разработчик)
- Выполнено кодирование (Разработчик)
- Заведена задача на покрытие unit-тестами (Разработчик)
- Тестировщик создает подзадачи в истории и приступает к разработке тестов (Тестировщик)
- Наполнение истории тестами в Аllure (Тестировщик)
- Выполнена проверка кода, разработчик истории перевел задачу в статус Review (Разработчик)
- Код просмотрен (Разработчик)
- Разработчик проверяет создал ли он ту ценность, которую хотела команда (Разработчик)
- В истории в комментариях указано описание изменений (Разработчик)
- Код прошел проверку внешними системами (GIT, ASC), изменения смержены в ветку main (Разработчик)
- История готова к тестированию, Разработчик переводит задачу в статус Ready for Testing (Разработчик)
- Проведено необходимое тестирование, с фиксацией результатов в трекере или системе автотестирования, результаты тестирования приложены, сделан регресс (Тестировщик)
- История протестирована и соответствует всем критериям приемки и тестовым сценариям (Тестировщик)
- Отсутствуют дефекты (major, critical, blocker) (Тестировщик)
- В коде сделано покрытие авто-тестов (Разработчик)
- Принята CPO или продемонстрирована на Демо (СТО)
- Сформирована минимально необходимая документация release notes (включена в релиз) (СТО)
- Загружена на прод стенды (Install)
- CTO закрывает историю (Closed)
